Before writing code...

Read these documents in order.

1. WHY.md
2. CONTEXT.md
3. README.md
4. docs/philosophy.md
5. docs/vision.md
6. docs/prd.md

Do NOT begin implementation until you understand them.

---

Before every commit ask yourself:

- [ ] Does this help filmmakers?
- [ ] Does it support storytelling?
- [ ] Does it reduce production friction?
- [ ] Does it fit the production workflow?
- [ ] Does it hide technical complexity?
- [ ] Does it preserve the Single Source of Truth?
- [ ] Does it keep AI as the crew?
- [ ] Does it make the project simpler?
- [ ] Does it improve long-term architecture?

If any answer is "No"

Do not code.

Redesign.

---

Before opening a Pull Request

- [ ] Documentation updated
- [ ] Architecture considered
- [ ] Tests pass
- [ ] Existing workflows unchanged
- [ ] No duplicated functionality
- [ ] No breaking production workflow
